Static Website Hosting on AWS

A serverless and scalable solution

Step 1: DNS Lookup (Route 53)

The user's browser performs a DNS lookup for the website's domain (e.g., example.com) which is managed by AWS Route 53. Route 53 directs the traffic to CloudFront.

Step 2: Access Website (HTTPS)

The user's request reaches CloudFront, which enforces secure HTTPS communication for the entire session.

Step 3: Use SSL Certificate (ACM)

CloudFront uses an SSL certificate, provisioned and managed by AWS Certificate Manager (ACM), to establish a secure connection with the user's browser.

Step 4: Request from Origin (CloudFront)

CloudFront, acting as a CDN, checks its cache for the requested file. If the file is not in the cache, it forwards the request to the origin, which is an S3 bucket.

Step 5: Redirect Requests (S3)

The request is handled by an S3 bucket. A private S3 bucket is configured to redirect all incoming requests to a public S3 bucket that contains the static website files.
